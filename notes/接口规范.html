二、数据库表结构设计建议（v3 补充项）

现有 v3（10张表）已满足基本需求。以下为前端 Dashboard 所需的 追加建议。

① stores 表新增字段
-- 区分老店/新店 + 开业日期（用于 Sheet3 新店分析） ALTER TABLE stores ADD COLUMN store_type TEXT DEFAULT '老店'; -- '老店' | '新店' ALTER TABLE stores ADD COLUMN open_date TEXT; -- 格式 YYYY-MM，如 '2025-02'（铁机路店） -- 更新预置数据 UPDATE stores SET store_type='新店', open_date='2025-01' WHERE store_id='MD00012'; -- 蔡甸中百 UPDATE stores SET store_type='新店', open_date='2025-02' WHERE store_id='MD00017'; -- 铁机盛世家园
② 建议新增索引（性能优化）
-- 月度汇总常用查询 (store_id + month + channel) CREATE INDEX IF NOT EXISTS idx_ms_store_month ON monthly_summary(store_id, month, channel); -- 品项统计按月+门店查询 CREATE INDEX IF NOT EXISTS idx_iss_store_month ON item_sales_summary(store_id, month, category); -- 时段统计常用组合 CREATE INDEX IF NOT EXISTS idx_ts_store_month ON timeslot_summary(store_id, month, channel); -- category_mapping 快速查标准名 CREATE INDEX IF NOT EXISTS idx_cm_raw ON category_mapping(raw_name, store_id);
③ 建议新增：年度汇总视图（Sheet1/2 同比环比）
-- 方便前端一次查询得到双年数据，无需两次请求 CREATE VIEW IF NOT EXISTS v_annual_compare AS SELECT store_id, month, SUBSTR(month, 1, 4) AS year, SUBSTR(month, 6, 2) AS mon, channel, total_amount, total_income, total_discount, order_count AS tc, avg_order_amount AS ac FROM monthly_summary;
④ 建议新增：品类月度透视视图（Sheet6）
-- 直接返回标准品类+月份组合，前端无需再 JOIN CREATE VIEW IF NOT EXISTS v_category_monthly AS SELECT iss.store_id, iss.store_name, iss.month, COALESCE(cm.standard_name, iss.category) AS std_category, SUM(iss.total_quantity) AS qty, SUM(iss.total_amount) AS amount, SUM(iss.total_income) AS income FROM item_sales_summary iss LEFT JOIN category_mapping cm ON cm.raw_name=iss.category AND cm.store_id IS NULL GROUP BY iss.store_id, iss.month, COALESCE(cm.standard_name, iss.category);

数字格式规范
  • 营业额 ≥ 10000：显示为 X.Xx万，如 5.2万
  • 营业额 < 10000：显示为 X,XXX 元
  • TC：整数，如 3,036
  • AC：保留1位小数，如 ¥12.8
  • MoM/YoY%：±X.X%，正绿负红，如 +5.3%
  • 优惠率：X.X%，如 5.5%
  • 占比：X.X%，如 28.9%

  • 指标：Tab（营业额/TC/AC）
  • URL 同步：筛选状态写入 query string（可分享链接）
  • 筛选记忆：localStorage 记录上次选择
