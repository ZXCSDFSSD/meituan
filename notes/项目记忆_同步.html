Excel 实际格式（v2 已验证）

  • 前 2 行是标题和筛选条件，第 3 行（range: 2）才是真正表头
  • 末尾有"合计"汇总行，过滤条件：storeName === '合计' || storeName === '--'
  • 三个文件各一个 Sheet（不是多 Sheet 合一文件！）：
    • 全渠道订单明细 → Sheet "全渠道订单明细"
    • 菜品销售明细 → Sheet "品项销售明细"
    • 收款明细 → Sheet "收款明细"

实际 Excel 关键列名（v2 已验证）

全渠道订单明细（无机构编码！）

  • store_name ← 门店名称 | order_id ← 订单号
  • order_amount ← 订单金额（无"元"后缀）| customer_paid ← 顾客实付
  • order_income ← 订单收入 | order_discount ← 订单优惠
  • channel来源 ← 订单来源 | dining_type ← 用餐方式
  • created_time ← 创建时间 | completed_time ← 完成时间

菜品销售明细（store_id 唯一来源）

  • store_id ← 机构编码 | dish_name ← 品项名称
  • amount ← 销售金额(元)（半角括号）| income ← 品项收入(元)
  • discount ← 优惠金额(元) | dish_category ← 菜品大类
  • dish_sub_category ← 菜品小类 | dish_ordered_time ← 点菜时间
  • meal_period ← 下单时间所属餐段 | order_source ← 订单来源

收款明细（无机构编码！）

  • store_name ← 门店 | payment_method ← 结账方式
  • payment_amount ← 支付金额(抵扣金额) | payment_discount ← 支付优惠
  • received_amount ← 收款金额 | net_amount ← 到账金额
  • handling_fee ← 手续费(服务费) | biz_sub_type ← 业务小类

数据库表（v3，10张）

  • stores：门店信息（PK: store_id）+ location_type + has_partner（v3新增）
  • orders：订单主表（PK: order_id）← 全渠道订单明细
  • order_dishes：品项明细（无 FK 约束）← 菜品销售明细；v3新增 dish_sub_category/dish_ordered_time/meal_period/order_source
  • sales_summary：日度统计（UNIQUE: store_id+date+channel）← 聚合自 orders
  • monthly_summary：月度统计（UNIQUE: store_id+month+channel）← 聚合自 orders
  • item_sales_summary：品项统计（UNIQUE: store_id+item_id+date）← 菜品销售明细聚合
  • payment_method_summary：收款汇总（UNIQUE: store_name+date+payment_method+biz_sub_type）
  • timeslot_summary：时段统计（UNIQUE: store_id+date+hour+channel）← 品项明细点菜时间，hour 0-23
  • category_mapping：品类名称标准化映射（raw_name+store_id → standard_name）
  • order_payments 表存在但目前不写入

store_id 映射（关键）

  • 全渠道订单明细 和 收款明细 均无机构编码，只有 门店名称
  • 只有 菜品销售明细 有机构编码 → 先处理写入 stores 表
  • 处理顺序必须：菜品销售明细 → 全渠道订单明细 → 收款明细
  • buildStoreNameMap() 按 store_name 查 store_id
