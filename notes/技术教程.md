## 1. TypeScript — JS 的升级版

### TypeScript 是什么？
TypeScript（简称 TS）是在 JavaScript 基础上加了**类型系统**的语言。
浏览器不能直接运行 TS，需要先编译（tsc）成 JS。

**类比**：JS 像便签纸（随便写），TS 像合同（每个字段都有格式要求）。

### 为什么用 TypeScript？

| 问题（纯 JS） | TypeScript 的解决方案 |
|-------------|---------------------|
| 调用 `data.total_amount` 但 `data` 可能是 null | 类型检查，编译时报错 |
| 函数参数传错了类型 | IDE 实时红线提示 |
| 团队协作不知道 API 返回什么格式 | interface 类型定义，自动补全 |

### 核心语法

```typescript
// 1. 基本类型标注
const name: string = '美团'
const count: number = 42
const visible: boolean = true

// 2. interface — 定义对象的"形状"
interface StoreInfo {
  store_id: string        // 必填字段
  store_name: string
  has_partner?: boolean   // ? 表示可选字段
}

// 3. 联合类型 — 只能是其中一个值
type Channel = 'all' | '堂食' | '美团外卖' | '饿了么' | '京东秒送'

// 4. 泛型 <T> — 通用的类型参数
// useState<string>('') → 这个状态是 string 类型，初始值 ''
// Promise<number>      → 这个 Promise 成功后返回 number
const [count, setCount] = useState<number>(0)

// 5. 类型断言 as — 告诉 TS "我知道它的真实类型"
const value = someUnknown as string

// 6. 可选链 ?. — 安全访问可能为 null 的属性
const city = store?.city   // store 为 null 时返回 undefined，不报错

// 7. 空值合并 ?? — null/undefined 时的默认值
const name = store?.name ?? '未知门店'
```

### 本项目中的 TS 关键决策

- `ChartThemeConfig extends Record<string, unknown>` — 使 ECharts 主题对象可以被展开 (`...baseTheme`)
- `client.get<never, OverviewData>(url)` — 泛型参数给 Axios 加类型，GET 请求没有 body 用 never 占位

## 2. React — 构建用户界面的框架

### React 是什么？
React 是 Facebook 开源的 UI 库。核心思想：**把 UI 描述成函数**。

```
状态（State）→ 函数（组件）→ UI（HTML）
```

当状态变化时，React 自动重新调用函数，更新界面（不需要手动操作 DOM）。

**类比**：Vue/React 像会自动更新的电子屏幕；原生 JS 需要你手动说"把这块改成那个"。

### 函数组件

```tsx
// 最简单的函数组件
function Hello({ name }: { name: string }) {
  return <div>你好，{name}！</div>
}

// 使用：<Hello name="美团" />
// 渲染结果：<div>你好，美团！</div>
```

**JSX 语法**：在 JS 里直接写 HTML 标签（实际上是语法糖，编译成 React.createElement()）。

### useState — 状态管理

```tsx
import { useState } from 'react'

function Counter() {
  // [当前值, 更新函数] = useState(初始值)
  const [count, setCount] = useState(0)

  return (
    <div>
      <p>计数：{count}</p>
      {/* 点击时调用 setCount，React 自动重新渲染 */}
      <button onClick={() => setCount(count + 1)}>+1</button>
    </div>
  )
}
```

### useEffect — 副作用处理

```tsx
import { useState, useEffect } from 'react'

function StoreList() {
  const [stores, setStores] = useState([])

  // 组件挂载后执行一次（依赖数组为 []）
  useEffect(() => {
    fetch('/api/analytics/stores')
      .then(res => res.json())
      .then(setStores)
  }, [])  // ← 依赖数组：[] 只执行一次，[id] 当 id 变化时执行

  return <ul>{stores.map(s => <li key={s.store_id}>{s.store_name}</li>)}</ul>
}
```

**useEffect 执行时机**：
- `[]` → 只在组件挂载时执行一次（相当于 Vue 的 `mounted`）
- `[dep]` → 挂载 + dep 变化时执行（相当于 Vue 的 `watch`）
- 无依赖数组 → 每次渲染都执行（通常不用）

### useRef — 引用

```tsx
import { useRef } from 'react'

function MyChart() {
  // 绑定 DOM 元素
  const containerRef = useRef<HTMLDivElement>(null)

  // 保存实例（改变不触发渲染）
  const chartRef = useRef(null)

  useEffect(() => {
    // containerRef.current → 真实的 DOM 节点
    const chart = echarts.init(containerRef.current)
    chartRef.current = chart   // 保存实例，不触发重渲染
  }, [])

  return <div ref={containerRef} />
}
```

**useState vs useRef**：
- useState：存"要显示什么"→ 改变触发渲染
- useRef：存"工具/DOM/实例"→ 改变不触发渲染

### 自定义 Hook

```tsx
// 把"获取数据+loading+error"的逻辑封装成 Hook
function useStoreDetail(storeId: string) {
  const [data, setData] = useState(null)
  const [loading, setLoading] = useState(false)
  const [error, setError] = useState(null)

  useEffect(() => {
    if (!storeId) return
    setLoading(true)
    fetch(`/api/analytics/store-detail/${storeId}`)
      .then(res => res.json())
      .then(setData)
      .catch(e => setError(e.message))
      .finally(() => setLoading(false))
  }, [storeId])

  return { data, loading, error }
}

// 在组件里使用
function StoreDetail({ id }) {
  const { data, loading } = useStoreDetail(id)  // 一行搞定
  if (loading) return <Spin />
  return <div>{data?.store.store_name}</div>
}
```

## 3. Vite — 超快的前端构建工具

### Vite 是什么？

Vite 是新一代前端构建工具（相当于升级版的 Webpack），主要提供两个功能：
1. **开发服务器**：启动 `npm run dev` 后提供热更新（改代码浏览器自动刷新）
2. **生产构建**：`npm run build` 把 TS/TSX 代码打包成浏览器能运行的 JS

### 关键配置（vite.config.ts）

```typescript
export default {
  plugins: [react()],   // 支持 React/JSX
  resolve: {
    alias: {
      '@': '/src',       // @ 是 src/ 目录的别名
                         // import '@/api' = import '/src/api'
    },
  },
  server: {
    proxy: {
      '/api': 'http://localhost:3000',  // 代理：前端 /api/xxx → 后端 localhost:3000/api/xxx
    },
  },
}
```

**代理为什么重要？**

前端开发时运行在 `http://localhost:5173`，后端运行在 `http://localhost:3000`。
浏览器的跨域安全策略（CORS）会阻止 5173 直接请求 3000 端口。
Vite 代理让前端代码写 `/api/xxx`，Vite 帮你转发到后端 3000，解决跨域问题。

## 4. Ant Design — React 组件库

### Ant Design 是什么？

Ant Design（简称 AntD）是蚂蚁金服开源的企业级 React 组件库。
提供了几百个现成的组件（按钮、表格、表单、布局等），不需要从零写 CSS。

**类比**：AntD 就像乐高积木，每个组件是一块积木，拼起来就是界面。

### 核心概念

**ConfigProvider — 全局配置**
```tsx
<ConfigProvider
  locale={zhCN}   // 中文语言包
  theme={{
    algorithm: antdTheme.darkAlgorithm,  // 暗色主题
    token: { colorPrimary: '#4ECDC4' },  // 主色调
  }}
>
  <App />  {/* 所有子组件都会应用这个主题 */}
</ConfigProvider>
```

**栅格系统（Grid）**
```tsx
<Row gutter={[16, 16]}>          {/* 行，水平/垂直间距 16px */}
  <Col xs={24} sm={12} lg={8}>   {/* 列：手机全宽、平板半宽、桌面三分之一 */}
    <Card>内容</Card>
  </Col>
</Row>
```

AntD 把每行分成 **12格**（或 24格），通过 xs/sm/md/lg 控制在不同屏幕宽度下占几格。

**Table — 数据表格**
```tsx
const columns = [
  { title: '门店', dataIndex: 'store_name' },
  { title: '营业额', dataIndex: 'amount', render: (v) => `¥${v}` },
]

<Table
  dataSource={stores}      // 数据数组
  columns={columns}        // 列配置
  rowKey="store_id"        // 每行唯一key
  pagination={false}       // 不分页
/>
```

**Select — 下拉选择**
```tsx
<Select
  value={selectedChannel}
  onChange={(v) => setChannel(v)}
  options={[
    { value: 'all', label: '全渠道' },
    { value: '堂食', label: '堂食' },
  ]}
/>
```

### AntD v5 的 Design Token

AntD v5 引入了 Token 系统（类似 CSS 变量），设置 `token.colorPrimary` 后，
按钮、链接、选中状态、进度条等所有组件都会自动使用这个颜色，无需一一配置。

## 5. ECharts — 图表库

### ECharts 是什么？

Apache ECharts 是百度开源的强大图表库，支持柱状图、折线图、饼图、散点图等数十种图表。
我们使用 `echarts-for-react` 作为 React 封装版。

### 核心概念：option 对象

ECharts 通过一个大型 JSON 配置对象（option）描述图表：

```tsx
const option = {
  // 1. 提示框（鼠标悬停时弹出的信息框）
  tooltip: {
    trigger: 'axis',         // 鼠标碰到 X 轴线时触发
    formatter: (params) => { // 自定义内容（返回 HTML 字符串）
      return `${params[0].name}: ${params[0].value}元`
    },
  },

  // 2. X 轴
  xAxis: {
    type: 'category',        // 类目轴（用字符串标签）
    data: ['1月', '2月', '3月'],  // 标签数组
  },

  // 3. Y 轴（数组支持多个 Y 轴）
  yAxis: [
    { type: 'value' },       // 主Y轴（数值轴）
    { type: 'value' },       // 副Y轴（右侧）
  ],

  // 4. 图例（标注哪条线是什么）
  legend: { bottom: 0 },

  // 5. 数据系列
  series: [
    {
      name: '营业额',
      type: 'bar',           // 柱状图
      data: [8000, 9200, 7800],
      yAxisIndex: 0,         // 绑定到主Y轴
    },
    {
      name: '环比',
      type: 'line',          // 折线图
      data: [null, 15, -15], // null 表示该点断开
      yAxisIndex: 1,         // 绑定到副Y轴
    },
  ],
}

// 渲染
<ReactECharts option={option} style={{ height: 300 }} notMerge />
```

### 本项目中的图表类型

| 组件 | 图表类型 | 用途 |
|-----|---------|------|
| StoreTrendChart | 柱状图 + 折线图（双Y轴） | 营业额趋势 + 环比 |
| CategoryBarChart | 横向条形图 | 品类 Top8 |
| TimeslotAreaChart | 面积图 | 24小时时段分布 |
| DonutChart | 圆环图（饼图） | 渠道占比/收款方式 |

## 6. @antv/s2 — 多维透视表

### S2 是什么？

S2 是蚂蚁金服（AntV 体系）开源的**高性能多维表格**库。
专门处理"行、列、值三个维度"的交叉展示（类似 Excel 数据透视表）。

### 声明式 vs 命令式

| 风格 | 代表 | 说明 |
|-----|------|------|
| 声明式 | React/Vue/ECharts | 你描述"要什么"，框架帮你渲染 |
| 命令式 | S2/Canvas/Three.js | 你直接调用方法："创建→配置→渲染" |

### S2 的基本用法

```tsx
import { useEffect, useRef } from 'react'
import { PivotSheet } from '@antv/s2'

function PivotTable({ data }) {
  const containerRef = useRef(null)  // DOM 容器
  const s2Ref = useRef(null)         // S2 实例

  useEffect(() => {
    // 1. 数据配置
    const dataCfg = {
      fields: {
        rows: ['指标'],      // 行头维度
        columns: ['月份'],   // 列头维度
        values: ['数值'],    // 单元格内容
      },
      data: [
        { 指标: '营业额', 月份: '2025-11', 数值: '8.5万' },
        { 指标: '营业额', 月份: '2025-12', 数值: '9.2万' },
        { 指标: 'TC', 月份: '2025-11', 数值: '423' },
      ],
    }

    // 2. 渲染选项
    const options = {
      width: 600,
      height: 260,
    }

    // 3. 销毁旧实例（重要！避免内存泄漏）
    if (s2Ref.current) {
      s2Ref.current.destroy()
    }

    // 4. 创建新实例
    const s2 = new PivotSheet(containerRef.current, dataCfg, options)
    s2.render()
    s2Ref.current = s2

    // 5. 清理函数（组件卸载时自动调用）
    return () => { s2.destroy() }
  }, [data])  // data 变化时重新创建

  return <div ref={containerRef} />
}
```

## 7. Zustand — 全局状态管理

### 为什么需要全局状态？

当多个不同层级的组件需要共享同一份数据时：
- `useState` 只能在组件内部使用
- 通过 Props 层层传递（prop drilling）很麻烦
- Zustand 提供全局"仓库"，任何组件都能直接读取和修改

### Zustand vs Redux

| 特点 | Redux | Zustand |
|-----|-------|---------|
| 学习曲线 | 陡峭（Actions/Reducers/Dispatch） | 平缓（直接写函数） |
| 代码量 | 多 | 少 |
| 适用场景 | 大型复杂应用 | 中小型应用 |

### 基本用法

```typescript
import { create } from 'zustand'

// 创建 store
const useStore = create((set, get) => ({
  // 状态
  month: '2025-12',
  channel: 'all',

  // 修改状态的方法（action）
  setMonth: (m) => set({ month: m }),
  setChannel: (c) => set({ channel: c }),

  // 读取状态的方法
  getParams: () => {
    const { month, channel } = get()
    return { month, channel }
  },
}))

// 在组件里使用
function FilterBar() {
  const { month, setMonth } = useStore()
  return <Select value={month} onChange={setMonth} />
}

function Overview() {
  const month = useStore(s => s.month)  // 只读取 month，其他变化不触发重渲染
  return <div>当前月份：{month}</div>
}
```

### persist 中间件（本地持久化）

```typescript
import { create } from 'zustand'
import { persist } from 'zustand/middleware'

const useStore = create(
  persist(
    (set) => ({
      theme: 'dark',
      setTheme: (t) => set({ theme: t }),
    }),
    {
      name: 'my-store',  // localStorage 的 key
      partialize: (s) => ({ theme: s.theme }),  // 只持久化 theme
    }
  )
)
```

刷新页面后，`theme` 会从 localStorage 自动恢复。

## 8. Axios — HTTP 请求库

### Axios 是什么？

Axios 是 Node.js 和浏览器都能用的 HTTP 请求库，相当于功能增强版的 `fetch`。

### Axios vs fetch

| 功能 | fetch（原生） | Axios |
|-----|-------------|-------|
| 自动解析 JSON | ❌ 需要 `.json()` | ✅ 自动 |
| 错误处理（4xx/5xx） | ❌ 不会 reject | ✅ 自动 reject |
| 请求/响应拦截器 | ❌ 没有 | ✅ 有 |
| 超时设置 | ❌ 麻烦 | ✅ 一行配置 |
| 取消请求 | ❌ AbortController | ✅ cancelToken |

### 本项目的 Axios 封装

```typescript
import axios from 'axios'

// 创建实例（配置 baseURL 和超时）
const client = axios.create({
  baseURL: '/api',       // 所有请求都加这个前缀
  timeout: 15000,        // 15秒超时
})

// 响应拦截器：自动解包 data，统一错误格式
client.interceptors.response.use(
  (res) => res.data,  // 成功：返回 res.data 而不是 res
  (err) => {
    const msg = err.response?.data?.error || err.message || '请求失败'
    return Promise.reject(new Error(msg))
  }
)

// 使用
const data = await client.get('/analytics/overview?month=2025-12')
// data 直接就是后端返回的对象（不需要 .data）
```

### Promise 和 async/await

```javascript
// Promise 写法（链式调用）
fetch('/api/data')
  .then(res => res.json())
  .then(data => console.log(data))
  .catch(err => console.error(err))
  .finally(() => setLoading(false))

// async/await 写法（更像同步代码）
async function loadData() {
  try {
    const data = await fetch('/api/data').then(r => r.json())
    console.log(data)
  } catch (err) {
    console.error(err)
  } finally {
    setLoading(false)
  }
}
```

两种写法等价，`async/await` 更易读。

## 9. SQL — 数据库查询语言

### SQL 是什么？

SQL（Structured Query Language）用于操作关系型数据库（本项目用 SQLite）。

**类比**：数据库是仓库，表（Table）是仓库里的货架，SQL 是你给仓库管理员下的指令。

### 核心语法

```sql
-- 1. 查询（SELECT）
SELECT store_name, total_amount    -- 查哪些列
FROM monthly_summary               -- 从哪张表
WHERE month = '2025-12'            -- 过滤条件
  AND channel = 'all'
ORDER BY total_amount DESC         -- 按营业额降序排列
LIMIT 10;                          -- 只取前10行

-- 2. 聚合函数（统计）
SELECT channel,
       SUM(total_amount) AS total, -- 求和，起别名
       COUNT(*) AS rows,           -- 计数
       AVG(total_amount) AS avg    -- 平均
FROM monthly_summary
WHERE month = '2025-12'
GROUP BY channel;                  -- 按渠道分组（必须有GROUP BY才能用聚合函数）

-- 3. 多表连接（JOIN）
SELECT m.store_id, m.total_amount, s.location_type
FROM monthly_summary m             -- 主表，别名 m
LEFT JOIN stores s                 -- 连接门店信息表，别名 s
  ON m.store_id = s.store_id       -- 连接条件：store_id 相同
WHERE m.month = '2025-12';

-- 4. 条件表达式（CASE WHEN）
SELECT store_id,
       CASE WHEN order_count > 0
            THEN ROUND(total_amount / order_count, 2)  -- 计算客单价
            ELSE 0                                      -- 防止除以零
       END AS avg_ac
FROM monthly_summary;

-- 5. 安全参数化（? 占位符）
-- 代码里不要拼接字符串！用参数数组传值
db.query('SELECT * FROM stores WHERE store_id = ?', ['MD00001'])
```

### 本项目的数据库表结构

```
monthly_summary   ← 月度汇总（主要查询这张表）
  - store_id      门店ID
  - month         月份（YYYY-MM）
  - channel       渠道（all/堂食/美团外卖/饿了么/京东秒送）
  - total_amount  营业额
  - order_count   订单数
  - avg_order_amount 客单价

item_sales_summary ← 品项月度汇总
  - store_id      门店ID
  - month         月份
  - item_name     品项名称
  - category      品类
  - total_quantity 销售数量
  - total_amount  销售金额

timeslot_summary  ← 时段汇总
  - store_id      门店ID
  - month         月份
  - hour          小时（0-23）
  - channel       渠道
  - tc            订单数（Traffic Count）
```

## 10. Express.js — Node.js Web 框架

### Express 是什么？

Express 是 Node.js 最流行的 Web 框架，用于搭建 HTTP API 服务器。

**类比**：Node.js 是发动机，Express 是车身（提供路由、中间件等便利功能）。

### 基本结构

```javascript
const express = require('express')
const app = express()

app.use(express.json())  // 中间件：自动解析请求体中的 JSON

// 路由：GET /api/hello → 返回 JSON
app.get('/api/hello', (req, res) => {
  res.json({ message: '你好，世界！' })
})

// 路由：路径参数（:id）
app.get('/api/store/:id', (req, res) => {
  const storeId = req.params.id    // 从 URL 路径取值
  const month   = req.query.month  // 从 ?month=xxx 查询参数取值
  res.json({ storeId, month })
})

app.listen(3000)  // 监听 3000 端口
```

### 常见问题解答

**Q: 为什么有时候数据显示 "--"？**
A: 该门店/月份/渠道组合在数据库中没有记录。需要先通过数据导入（`node main.js --import`）加载 Excel 数据。

**Q: 修改了 TypeScript 代码后，浏览器自动刷新吗？**
A: 是的，Vite 的 HMR（热模块替换）会在保存文件后几百毫秒内自动更新浏览器，甚至保留当前状态（不全量刷新）。

**Q: `import '@/api'` 为什么能找到文件？**
A: `@` 是 `src/` 的路径别名，在 `vite.config.ts` 中通过 `resolve.alias` 配置。完整路径是 `src/api/index.ts`。

**Q: 怎么查看 API 返回的原始数据？**
A: 在浏览器开发者工具（F12）→ Network 标签页 → 找到对应请求 → Response 标签，或者直接在浏览器地址栏输入 `http://localhost:3000/api/analytics/overview`。
